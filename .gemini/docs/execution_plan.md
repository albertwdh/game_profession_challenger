# 平台策略与推进方案 (Execution Plan)

## 1. 平台选择 (Platform Strategy)
考虑到游戏的 2D 文字/数值驱动属性及独立开发背景，采用 **"Web-First, Cross-Platform Later"** 策略。

### 首选平台：Web (Mobile-friendly SPA)
- **原因：** 无需安装、即点即玩、易于在社交平台（微信、微博、小红书）分享链接。
- **技术：** React PWA (Progressive Web App)，可实现类似原生 App 的离线和桌面图标体验。

### 次选平台：PC (Steam/Itch.io)
- **原因：** 职场模拟类游戏在 Steam 上有稳定的受众群体。
- **技术：** 通过 Electron 包装 Web 版本即可。

### 移动端：微信小程序 / App
- **原因：** 适合碎片化时间。
- **技术：** React Native 或 uni-app (若后期需要原生体验)。

## 2. 推进阶段 (Phased Roadmap)

### 阶段一：MVP 验证 (第 1-3 周)
- **目标：** 跑通“生存 30 天”的核心闭环。
- **重点：** 文案梗的质量、数值平衡。
- **产出：** 一个可在浏览器运行的、包含 1 个职业（全栈开发）的 Demo。

### 阶段二：社区化与多职场 (第 4-6 周)
- **目标：** 增加重复可玩性。
- **重点：** 引入新媒体、萌新职业；加入“死法排行榜”。
- **产出：** 包含 3 个职业、有后端排行榜支持的 Beta 版本。

### 阶段三：商业化与分发 (第 7-10 周)
- **目标：** 流量变现。
- **重点：** 接入广告、优化社交分享卡片、Steam 商店页准备。

## 3. 宣发策略
- **内容营销：** 在小红书/V2EX/掘金发布“我做的游戏又让我猝死了”等槽点内容。
- **玩家共创：** 开放“事件投递”入口，让用户贡献真实的职场离谱事件。
