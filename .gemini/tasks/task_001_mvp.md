# Task 001: MVP 版本开发

## 1. 任务目标
构建一个包含核心循环的单机 Web 版本，支持玩家以“全栈开发”职业进行为期 30 天的职场生存挑战。

## 2. 设计规范 (Design)
- **数值设计:**
  - 初始值：存款 2000, 精神 100, 能力 10, 摸鱼 0。
  - 每日消耗：存款 -50 (生活费), 精神 -5 (基础压力)。
- **核心交互 - “叙事反馈”:**
  - **剧情回响 (Narrative Echo):** 每个选项点击后，显示一段充满职业梗的剧情反馈，解释为什么数值会这样变动。
  - **知识点掉落 (Career Insight):** 完成特定任务后，弹出“职场冷知识”或“避坑指南”，让玩家真实学到东西。
- **事件结构:**
  - **日常搬砖:** 枯燥但稳定的资源获取。
  - **奇葩突发:** 极具幽默感和挑战性的随机事件。
  - **连环事件:** 前期的选择会影响后期特定事件的触发概率或选项。

## 3. 开发路线 (Development)

### Step 1: 增强型数据管线
- 扩展 `events.json`：增加 `ReactionText`（反馈剧情）和 `Insight`（科普贴士）字段。
- 准备首批 20 个“有梗且有干货”的事件。

### Step 2: 交互引擎开发
- **StoryModal:** 开发一个专门处理剧情反馈和知识点展示的弹窗组件。
- **EventSystem:** 实现简单的状态标记（Flags），支持根据之前的选择改变后续事件。

### Step 3: 游戏逻辑实现
- 随机事件抽取算法（增加“剧情相关性”权重）。
- 结局判定系统：不仅根据数值，还根据玩家在重大分歧点上的累计倾向给出“职业评价”。

## 4. 测试策略 (Testing)
- **单元测试:** 针对数值结算逻辑进行边缘测试（如：存款恰好为 0 时的判定）。
- **平衡性测试:** 使用 Python 运行 1000 次模拟选择（随机选择 vs 极端选择），统计平均存活天数。
- **UI 测试:** 确保在手机浏览器和 PC 浏览器上的响应式适配。

## 5. 验收标准
- [x] 玩家能够顺利完成 1 天的完整循环。
- [x] 四大资源数值随选择正确更新。
- [x] 任何资源归零时能触发相应的 GameOver 界面。
- [x] 界面适配移动端，无明显样式错位。
- [x] **MVP 独立化**: 代码已隔离至 `mvp/` 文件夹，依赖已安装，可直接运行。
